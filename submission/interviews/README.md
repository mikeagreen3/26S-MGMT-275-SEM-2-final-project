# Interviews

Raw user interview notes for the UX & Eval Appendix. **8-12 total for the team** (8 is the minimum per the assignment spec). Split between Michael and Jessicya to be agreed.

## Public-repo PII policy

This repo is public (required by the assignment). Treat interview material as follows:

- **`interviews/raw/`** (gitignored) — full notes, real names, contact info, recordings. Stays local.
- **`interviews/` root** (committed) — anonymized synthesis only. Use a handle (e.g., "Host A — 3 properties, urban"). No real names, no email/phone, no property addresses, no platform-identifying screenshots.
- **Consent:** confirm with each interviewee that anonymized quotes may appear in a public academic repo before quoting them.

## Naming convention

Anonymized synthesis files: `host-<letter>-<short-descriptor>.md` — e.g., `host-a-3-props-urban.md`
Raw files (gitignored): `YYYY-MM-DD-<host-handle>.md` in `interviews/raw/`

## File template

Each interview file should include:
- Frontmatter: `type: interview`, `interviewer`, `date`, `duration`, `recording_url` (optional), `consent: recorded|notes-only`
- Background: host profile (properties, platform, length of hosting)
- JTBD probes: current access workflow, last failure, what they tried, what fixed it
- Reactions to concept: AI Access Concierge capabilities (booking sync, NL scheduling, pre-arrival check, failure diagnosis)
- Quotable moments
