# Host F Synthesis — The Yale Yale-er

## Anonymized profile

Suburban host with 2 units (third under contract), roughly six years hosting, Airbnb-only, solo operator with a long-tenured cleaner. Tech stack is Yale Assure locks plus August Connect on two doors per property. High tech savviness, opinionated, runs a heavy manual verification routine on top of the automation because they don't trust the sync.

## Top 3 pains (with quotes)

1. **Sync sending conflicting / wrong codes (Pain B dominant).** "The sync software malfunctioned and sent the guest two separate emails, each with a different access code, and didn't say which one was for which door."
2. **Keypad UX failure with guests.** "Guests don't know how to use a capacitive-ish keypad. They tap, they brush, they mash with one finger... I shouldn't have to make a tutorial video for a doorknob."
3. **Pre-arrival anxiety driving manual checking.** "I'd stop checking the August app eight times a day. That's the biggest thing."

## Reactions per capability

- **Pre-arrival readiness check:** Strongest pull. "The pre-arrival check, by a mile." Wants it earlier than 2 hours pre-arrival, ideally the evening before so they can drive over if needed.
- **Guest code lifecycle:** Table stakes, treated with skepticism. Yale tried this and bungled it, so they want to watch it work for several bookings before going hands-off.
- **Cleaner code rotation:** Highest-trust-cost feature. Wants a "rotation happened, confirm to commit" step for the first 30-60 days before letting it auto.
- **Ready message:** Push notification only, short, prefers a "silent if all clear" mode. "I need a red light when it matters."
- **Not-ready message:** Must be specific (which property, which door, which failure, what to do). "Don't make me guess."
- **No remote unlock:** Endorsed enthusiastically. They've disabled remote unlock on every device they've owned.

## Standout quotes

- "It's the thing I've been hand-rolling for six years, except automated."
- "I don't need a green light every single time. I need a red light when it matters."
- "If I have to keep checking anyway, you've sold me peace of mind and delivered the same anxiety with a higher monthly bill."
- "All-or-nothing is how you lose me at the door. One property is how you earn the second."

## Van Westendorp pricing

| Question | Price (per property / month) |
|---|---|
| Too expensive | $50 |
| Expensive but considering | $25 |
| Great deal | $10 |
| So cheap it's suspect | <$3 |

Anchored to per-property pricing, not per-account. Sensitivity is real but not extreme: they explicitly compared the $10 great-deal price to the cost of a single lost-star review.

## Killer objection

"I already own four Yale locks I paid good money for and your software doesn't run on them. Why is this my problem to solve, not yours?" Hardware switching cost is the single hardest thing for a Schlage rep to answer with this persona. They explicitly ask for a Yale-compatible version or a trade-in program before they'll seriously consider switching.

## What surprised us

- They endorsed every anti-goal, including the no-guest-messaging one we expected pushback on. Their reasoning was protective of their Airbnb review channel ("Don't put a robot between me and my review score"), not deference.
- They wanted the readiness check to run earlier than 2 hours pre-arrival. The current spec window may be too close to check-in for hosts who'd actually drive to the property to fix something.
- They volunteered that they'd prefer a silent-on-success mode for the ready message, which inverts the default assumption that hosts want a "you're good" confirmation each time.
- The keypad physical-feel concern came up unprompted three times. For this persona, the readiness software is necessary but not sufficient. Hardware feel is a gating decision.

## Trial-property signal

**Strong one-property trial signal.** When asked directly, they said unambiguously: "One property. Definitely one." They'd put Schlage on the smaller unit for a season (two or three guest cycles), evaluate whether the readiness check actually catches failures before they happen, and only swap the second property the next off-season if the trial succeeded. They were emphatic that an all-or-nothing pitch loses them at the door. Implication for go-to-market: a single-lock pilot offer (one property, one season, easy return / trade-in if it doesn't perform) is the unlock for multi-property Yale incumbents.
